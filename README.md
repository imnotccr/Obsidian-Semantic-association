# 语义关联（Semantic Connections）

一个用于 **Obsidian（桌面端）** 的语义索引插件：为你的 Vault 建立语义索引，并在右侧展示“与当前笔记最相关的其他笔记”。当前阶段优先维护关联视图、索引管理与变更同步；语义搜索（自然语言检索）暂列为后续待办。

> 注意：本插件会把笔记内容（按段落切分后的文本）发送到你配置的 **远程 Embeddings 接口** 来生成向量，请务必确认你的数据合规与隐私要求。

>以及本项目完全依靠AI生成,作者本人也在学习,对于一些使用上的问题无法进行解决,测试模型使用的是BAAI/bge-m3,可在硅基流动上免费使用

> 当前范围说明：仓库中仍保留语义搜索相关代码路径，但它目前不作为正式支持能力，也不在本阶段处理范围内，后续再统一整理。

## 主要功能

- **右侧关联视图**：在右侧侧栏展示与当前笔记最相关的笔记，并显示“最契合段落”预览；点击可跳转并高亮对应行范围。
- **语义搜索（待办）**：自然语言检索能力暂列后续事项，当前版本不作为正式支持范围。
- **索引管理**：手动全量重建索引、查看索引存储统计、重试网络/429 等导致的失败索引任务。
- **变更同步**：可选择开启**自动增量索引**，仅对新增/修改/删除/重命名涉及的笔记自动更新索引；也可手动执行“同步变动笔记”做一次补扫。

## 快速开始（如何使用）

### 1) 安装插件

把插件文件放到你的 Vault 插件目录并启用：

- 目标目录：`{vault}/.obsidian/plugins/semantic-connections/`
- 需要的文件：
  - `main.js`
  - `manifest.json`
  - `styles.css`

如果你是从源码构建，再拷贝 `dist/` 下的同名文件即可（见文末“开发与构建”）。

### 2) 配置远程 Embeddings

进入 Obsidian 设置 → 插件设置 → **语义关联** → **远程嵌入**：

- **API 基础 URL**：只需填写服务根地址（例如 `https://api.example.com`）；插件会自动补齐为 `{baseUrl}/v1/embeddings`，如果你粘贴了包含 `/v1` 或 `/v1/embeddings` 的完整地址，也会自动规范化。
- **API 密钥**：作为 `Authorization: Bearer ...` 发送。
- **远程模型**：默认 `BAAI/bge-m3`。
- （可选）**超时时间**（毫秒）、**批大小**（单次请求的最大文本条数）。
- 点击 **测试连接**：成功后会显示向量维度（dimension）。

### 3) 第一次建立索引（必须）

插件**不会在启动时自动全量重建**（避免意外的远程 API 消耗）。首次使用请手动执行：

- 命令面板：`重建索引`（command id：`rebuild-index`），或
- 设置页：**索引管理** → `重建`

索引完成后，右侧视图才会有结果。

### 4) 打开视图

命令面板中执行：

- `打开关联视图`（`open-connections-view`）：右侧展示“语义关联”

语义搜索相关入口当前先不作为正式使用流程，后续会在能力和交互稳定后再重新纳入文档。

你也可以在设置里开启 **启动时自动打开右侧关联视图**。

## 日常使用建议

### 关联视图怎么读

- 列表的“相关度(%)”= `原始分值 × 100`（原始分值通常在 -1～1 之间）。
- 每条结果会展示该候选笔记中与当前笔记**最契合的一段**作为预览；点击即可跳转并高亮。
- **相关度阈值**是软过滤：无论阈值多高，仍会展示最相关的前 5 条，并将低于阈值的结果标为“弱关联”。

### 保持索引更新：自动增量索引 / 同步变动笔记

设置项 **自动增量索引** 开启后：

- 新建/修改笔记时，插件会在停止编辑约 1 秒后自动做 hash 检查，并仅对发生变化的笔记执行增量索引。
- 删除/重命名笔记时，插件会自动清理或迁移对应索引数据。
- 如果你希望对整个 Vault 再做一次全面检查，仍可手动执行以下同步（会调用远程 Embeddings API）：
  - 命令：`Sync Changed Notes（同步变动笔记）`（`sync-changed-notes`）
  - 或在关联视图顶部提示条点击 `[立即同步]`

### 什么时候需要“重建索引”

建议在以下情况手动全量重建：

- 第一次使用或索引为空
- 你更换了 `API 基础 URL` / `远程模型` / 向量维度发生变化
- 你调整了排除文件夹，且希望把被排除的内容从索引存储中彻底移除
- 错误日志提示索引快照不兼容

## 设置项说明

- **排除文件夹**：每行一个路径（如 `templates`），索引时跳过；搜索/关联也会立即过滤这些路径下的结果。
- **行为**
  - 启动时自动打开右侧关联视图：启动完成后在右侧创建视图（不抢编辑器焦点）
  - 自动增量索引：监听文件变更，并自动更新受影响笔记的索引
- **远程嵌入**
  - API 基础 URL / API 密钥 / 远程模型
  - 超时时间（ms）/ 批大小
  - 测试连接：发送一次真实 embeddings 请求
- **关联视图**
  - 相关度阈值：`0.0–1.0`
  - 每篇笔记最多展示段落数：`0` 表示不限制
- **索引管理**
  - 重建：全量重建索引并保存快照
  - 存储统计：查看索引快照的磁盘占用
  - 重试失败项：重试网络中断或 429 限流导致的索引失败任务

## 相关度是如何计算的（简述）

- 插件将笔记切分为多个 chunk，并为每个 chunk 生成 embedding 向量。
- 检索阶段使用 **余弦相似度（cosine similarity）** 作为分值：
  - `score = dot(q, v) / (||q|| * ||v||)`
- 在“关联视图”中，最终排序分数采用“最强关联片段”的相似度（让 UI 展示的片段与排序口径一致）。

## 数据文件与日志在哪里

插件运行时数据由 Obsidian 写入：

- `{vault}/.obsidian/plugins/semantic-connections/`
  - `data.json`：插件设置
  - `index-store.json`：索引快照（结构化元数据）
  - `index-vectors.bin`：向量快照（Float32 二进制）
  - `error-log.json`：错误日志
  - `runtime-log.json`：运行日志
  - `failed-tasks.json`：可重试的失败任务

插件不会改写你的 `.md` 笔记文件。
